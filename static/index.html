代码输入
支持三种代码提交方式，灵活适配不同开发场景。粘贴代码片段进行快速审查，上传本地文件或指定目录路径进行批量扫描，也可直接提供 GitHub 仓库 URL 自动克隆全仓库代码。
GitHub 仓库本地路径粘贴代码文件上传
🔍
代码解析与预处理
自动识别超过 30 种代码文件类型（Python、JavaScript、Go、Java、Rust 等），过滤 node_modules、.git、__pycache__ 等非代码目录，将所有文件拼接为带路径头标识的统一审查输入，并校验总大小不超过 200KB 上限。
文件识别目录过滤大小校验拼接合并
🔬
静态分析
运行 flake8 和 pylint 对代码进行静态检查，预先捕获语法错误、代码风格违规、未使用变量、导入顺序等基础问题。静态分析结果将作为辅助上下文提供给后续的智能体审查阶段，不重复标记已有问题。
flake8pylint语法检查风格扫描
🤖
AgentTeam 并行审查
基于 LangGraph 编排引擎，三个专业 AI 智能体同时启动，各自独立审查代码的不同维度：
🔐
安全审查专家
检测注入漏洞（SQL / 命令 / LDAP）、XSS 跨站脚本、硬编码密钥与密码、认证授权缺陷、不安全的加密算法、路径遍历等
7 个审查维度
⚡
性能优化师
分析算法复杂度（O(n²) 检测）、N+1 数据库查询、内存泄漏风险、不必要的 IO 操作、缓存策略缺失、并发与锁竞争问题
7 个审查维度
🧠
业务逻辑审核
审查命名规范与代码可读性、函数设计合理性、圈复杂度、DRY 原则遵守、错误处理完备性、设计模式适用性及可测试性
8 个审查维度
🎯
决策者汇总去重
Orchestrator 智能体收集三个专家的全部 findings，执行智能去重合并——当多个专家发现同一问题时合并为一条记录并标注发现来源。随后进行严重等级校准（critical / high / medium / low），按实际影响和优先级排序，确保最重要的安全问题排在最前面。
去重合并等级校准优先级排序交叉验证
🔧
生成修复方案
根据汇总后的 findings 自动生成完整的修复后代码和 Unified Diff 对比。左侧展示原始代码、右侧展示修复后代码，逐行标注增删改。修复代码保持原有功能逻辑不变，仅针对标记的问题进行精准修复。无法自动修复的问题在说明中标注原因。
修复代码Unified Diff左右对比逐行标注
📊
报告持久化存档
审查报告自动存入 SQLite 数据库，包含完整的安全审查结果、性能分析、质量评估、合并报告及修复代码。支持按时间倒序浏览历史记录，随时回溯查看任意历史审查的详细报告和代码对比，便于追踪代码质量演变趋势。
SQLite 存储历史回溯报告查看趋势追踪
